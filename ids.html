Sistemas de detección de intrusiones (IDS)

Estos sistemas permiten identificar y, posteriormente, responder ante actividades no legítimas en recursos de la red o un sistema mediante mecanismos automatizados. De esta forma, se logra un mayor nivel de seguridad en las transferencias de información entre cada una de las unidades de una organización.

Los principales componentes de un IDS son:

  • Sensores basados en equipo (host based sensors o HIDS). Reciben la información de aplicaciones en ejecución en un host en particular.
  • Sensores basados en red (network based sensors o NIDS). Recolectan información del tráfico de red.
  • Procesadores de eventos. Reciben la información procedente de los sensores y realizan inferencias sobre las intrusiones usando algún esquema:
    • Modelo de detección de usos indebidos: analizan el tráfico y lo comparan con una base de conocimiento (firma) sobre el modus operandi de agentes maliciosos.
    • Modelo de detección de anomalías: analizan el tráfico e intenta identificar actividad sospechosa, comparándolo con un perfil de comportamiento de usuario, proceso y servicio recurrente.

El conocimiento acerca de un modelo de secuencias y actividades maliciosas se denomina firma. Los métodos para examinar el tráfico de red son:

  • Patrones de tráfico de red.
  • Análisis de protocolos de red.
  • Coincidencia de firma.

Entre algunos ejemplos de IDS tenemos:

  • SolarWinds Security Event Manager
  • FortiSIEM
  • Bro
  • Snort
  • OSSEC
  • AIDE
